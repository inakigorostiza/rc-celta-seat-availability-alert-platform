01 · 🎯 Introducción

El problema, la solución y por qué importa

Un aficionado quiere entrar a ver al Celta, la grada está llena, y cuando días después se libera un asiento, la taquilla no sabe que él lo quería. Esa memoria faltante es una pérdida de ingresos silenciosa.

❌
El problema

Demanda perdida invisible

Una parte significativa de los abonos y entradas que se liberan (devoluciones, pagos caídos, pre-asignaciones canceladas) nunca llega a manos de los fans que las querían. La venta se evapora porque nadie les avisó.

✅
La solución

Recuperación automatizada

Capturar el interés mientras no hay entradas, detectar continuamente la disponibilidad en ONEBOX, y enviar un email personalizado con deep-link de compra en el momento exacto en que se libera la grada solicitada.

💡

Para recordar

Recuperar demanda perdida es un negocio de timing. El valor no está en tener la lista: está en llegar al cliente en el instante en que el problema que le impedía comprar se ha resuelto.

04 · 🔑 Credenciales

Cuatro credenciales, ninguna cuesta dinero en plan inicial

Este es el setup mínimo. Todas las claves tienen plan gratuito generoso o free-forever. Reserva 30 minutos para tenerlas todas listas antes de empezar.

Qué necesitas

client_secret
Cadena de 32 caracteres que identifica tu canal en ONEBOX.
channel_id
Número entero que identifica el canal dentro de la organización (ej. 2287).

Dónde se pide

Las facilita el equipo técnico de ONEBOX. Pídeselas a tu contacto de cuenta.

⚠️

Cuidado

El client_secret da acceso a todo el catálogo del canal. Nunca lo metas en el frontend ni en un repositorio público. Guárdalo en el gestor de secretos de tu plataforma.

Qué necesitas

API key
Token para llamar a la Marketing API desde n8n y la automatización.
Audience ID
Identificador del "audience" (lista) donde irán tus contactos.
Server prefix
Sufijo de la API key (ej. ...-us1 → us1). Indica el datacenter donde vive tu cuenta.

Dónde encontrarlo

API key:  Profile  →  Account & billing  →  Extras  →  API keys

Audience ID:  Audience  →  Settings  →  Audience name and defaults

Qué necesitas

API key
Token para llamar a la Transactional API desde n8n.
Dominio de envío verificado
Un dominio tuyo con DKIM + SPF publicados y validados en Mandrill.

Dónde encontrarlo

API key:  Mandrill  →  Settings  →  SMTP & API Info  →  New API key

Dominios:  Settings  →  Sending Domains
⚠️

Cuidado

Sin dominio verificado, Mandrill rechaza todos tus envíos con unsigned. Es el error nº 1 de los que empiezan. Publica los TXT records antes de montar el flow.

Qué necesitas

Measurement ID
Identificador G-XXXXXXX del web stream.

Dónde encontrarlo

GA4  →  Admin  →  Data streams  →  Web  →  tu stream

Este ID sí puede ir en el HTML público: está diseñado para ser visible. GA4 distingue sesiones mediante cookies y parámetros de URL, no por el ID en sí.

05 · 🎟️ ONEBOX

La API de ticketing

ONEBOX TDS es la plataforma donde el RC Celta vende sus entradas online. Expone una REST API profesional que nosotros usamos exclusivamente en modo lectura — jamás modificamos datos.

Modelo de datos (con nombres que entiende marketing)

Partido (ej. «Celta – Athletic · Jornada 29») └── Sesión (un partido concreto, con fecha y hora) ├── Tipos de precio (General, Junior, Abonado…) └── Gradas (Pista B1, Río Alto, Marcador…) └── disponibilidad = N asientos libres
💡

Para recordar

Al aficionado le hablamos de partido y grada. Los IDs internos (session_id, grada_id) son para las máquinas.

Los tres endpoints que usamos

POST /oauth/token Pedir un token de acceso (válido 12 horas)
GET /catalog-api/v1/sessions Listar todos los partidos del canal
GET /catalog-api/v1/sessions/{id}/availability Disponibilidad por grada de un partido

🔗 El enlace de compra (deep-link)

Cuando enviamos el email de aviso, el botón Comprar entrada apunta a una URL que abre ONEBOX directamente en el mapa de butacas, con los UTMs necesarios para que GA4 atribuya la conversión:

https://tickets.oneboxtds.com/rccelta/select/{sessionId}?    viewCode=V_blockmap    &utm_source=avisame    &utm_medium=email    &utm_campaign=recovery-app    &utm_content=event-{eventId}-grada-{gradaId}

• viewCode=V_blockmap es funcional (abre el mapa directamente).
• Los cuatro utm_* permiten a GA4 atribuir la compra al email concreto que generó el clic.

06 · 🌐 Landing Page

Estática, brandeada y conectada a ONEBOX

Una página HTML estática que imita el look & feel de las landing oficiales del RC Celta. Los datos del partido (imagen, fecha, gradas disponibles) se incrustan al compilar, leyéndolos en directo de ONEBOX.

⚡
Qué es

HTML estático · GitHub Pages

Una página generada en build-time a partir de un template Node.js. Se sirve directamente desde GitHub Pages con HTTPS automático.

🏗️
Por qué estática

Menos partes = más estable

Sin servidor en runtime, sin bases de datos en el hot path. Gratis de alojar. Trivial de cachear en CDN.

Flujo de envío del formulario

1
Usuario rellena

Nombre, email, grada, consentimientos.

2
Browser captura UTMs

Fuente, campaña, idioma, user-agent.

3
Envío a automatización

Validación + upsert a Mailchimp.

4
Thank-you + GA4

Evento generate_lead.

💡

Para recordar

El navegador nunca habla directamente con Mailchimp. Una automatización intermedia permite validar datos, aplicar lógica y mantener las API keys fuera del cliente.

07 · 📬 Mailchimp

La lista central, segmentable sin SQL

Aquí vive cada fan registrado. Dos mecanismos distintos organizan sus datos: merge fields (su ficha) y tags (su historial de intereses). Cada uno tiene una semántica diferente y eso cambia cómo los usamos.

🗂️ Merge fields Sobrescriben

La «ficha» del contacto. Cada campo guarda un solo valor: el más reciente. Ideal para datos que son por naturaleza únicos por contacto.

Ejemplo: Ana firma por Pista B1 hoy. Mañana firma por Río Alto.

GRADA_NAME: Pista B1 → Río Alto
GRADA_ID: 224549 → 224562

🏷️ Tags Se acumulan

Preservan todo el historial. Un contacto puede llevar decenas de tags simultáneamente. Ideal para segmentar por intereses o comportamientos.

Mismo caso: Ana firma por Pista B1 y luego por Río Alto.

grada:224549 (Pista B1)
grada:224562 (Río Alto)
event:4587
source:avisame
💡

Para recordar

Para segmentar usa tags. Para la ficha visible en el panel de Mailchimp usa merge fields. Son complementarios, no alternativos.

Segmento típico: «Avisar al abrir Sector 316 del Celta–Athletic»

tag contains  grada:224515         Y
tag contains  event:4587           Y
tag NOT ∋     notified:event-4587-grada-224515

Marketing construye este segmento en la UI de Mailchimp. n8n lo consume para enviar el aviso una sola vez por fan y grada.

08 · ✉️ Mandrill

Transaccional, no masivo

Mailchimp es para campañas programadas a listas grandes. Mandrill es para envíos inmediatos, personalizados, con analítica individual de aperturas y clics.

🎨 Anatomía del email

RC Celta · Entradas ¡Avísame!

¡Hay entradas!

Hola Ana, se acaban de liberar localidades para la grada que estás siguiendo. Corre que vuelan 🏃

Partido Celta – Athletic · Jornada 29
Grada Pista - B1
Disponibilidad 83 localidades
Comprar entrada
Te registraste en avisame.rccelta.es · No garantiza reserva

📊 Qué se mide por envío

Tracking activado

Open rate · CTR

Mandrill pixel de apertura + redirect-wrapping para clics. Se ve en su panel nativo y en GA4 via UTMs.

Filtros en el panel

Tags por partido + grada

Cada envío lleva avisame, event-{id}, grada-{id}. Vistas cruzadas en Mandrill Analytics.

⚠️

Cuidado

El dominio desde el que envías (from_email) debe estar verificado en Mandrill con DKIM + SPF. Si no, todos los envíos se rechazan con reject_reason: unsigned. Es el primer error que todo el mundo comete.

  • Workflow de n8n (con secretos sustituidos)

12 · ⚖️ Legal & Privacy

RGPD · LOPDGDD · LSSI-CE

Tres normativas se cruzan en este proyecto. Todas son obligatorias si operas en España. No son "nice to have" — sin cumplirlas, la plataforma no puede ir a producción.

🇪🇺
RGPD
Reglamento General de Protección de Datos · UE 2016/679

Normativa europea que establece el marco general sobre cómo se deben recoger, tratar y almacenar los datos personales. Aplica a cualquier tratamiento de datos de residentes de la UE.

  • Consentimiento libre, específico, informado e inequívoco antes de recoger el email
  • Minimización de datos — solo lo estrictamente necesario
  • Limitación de finalidad — usar los datos solo para lo que se consintió
  • Derechos ARCO-POL garantizados contractualmente
  • Registro de actividades de tratamiento documentado
🇪🇸
LOPDGDD
Ley Orgánica 3/2018

Adaptación española del RGPD. Refuerza los derechos de los usuarios y define el régimen sancionador nacional.

  • Edad mínima 14 años para consentir por sí mismos
  • Derechos digitales: portabilidad, olvido en redes, desconexión digital
  • Sanciones hasta 20 M€ o el 4 % de la facturación global
  • DPO obligatorio en tratamientos a gran escala
📧
LSSI-CE
Ley 34/2002 · Servicios de la Sociedad de la Información

Regula el envío de comunicaciones comerciales electrónicas y el uso de cookies.

  • Opt-in previo y expreso para emails comerciales (art. 21)
  • Enlace de baja gratuito y sencillo en cada comunicación
  • Identificación clara del remitente
  • Banner de cookies bloqueante para cualquier cookie no estrictamente necesaria (GA4 entra aquí)
  • Sanciones hasta 150 000 € por infracción grave
