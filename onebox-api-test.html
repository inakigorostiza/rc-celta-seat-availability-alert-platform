2. Get Available Sessions
Retrieve all available match sessions for the channel. Returns metadata and paginated list of matches with event details, venue information, and pricing.
GET
https://api.oneboxtds.net/catalog-api/v1/sessions
REQUEST
Headers
"Authorization": "Bearer " + [token from step 1]
Query Parameters (optional)
limit=100 (default) offset=0 (for pagination)
RESPONSE 200 OK
{ "metadata": { "limit": 100, "total": 21, "offset": 0 }, "data": [ { "id": 240895, "name": "NUMERADO", "type": "SESSION", "event": { "id": 4587, "name": "* API NUMBERED EVENT" }, "venue": { "id": 14, "name": "Estadio Saprissa", "location": { "city": "Barcelona", "country": "España" } }, "date": { "start": "2020-02-12T21:00:00+01:00" }, "price": { "min": { "value": 10 }, "max": { "value": 50 } } } ] }
3. Get Seat Availability
Retrieve available seats and pricing for a specific match session, organized by stadium section (grada). Use a session ID from the previous endpoint.
GET
https://api.oneboxtds.net/catalog-api/v1/sessions/{sessionId}/availability

Path Parameters
sessionId="240895" (from sessions endpoint)
Headers
"Authorization": "Bearer " + [token from step 1]

📋 Integration Notes

Authentication Flow: All API calls require a valid bearer token obtained from the authentication endpoint. Tokens expire after 12 hours and must be refreshed for long-running processes.

Session Management: Use the sessions endpoint to get all available matches. Each session has a unique ID required for fetching availability data. Sessions include pricing, dates, venues, and event metadata.

Availability & Grads: The availability endpoint returns seat information organized by stadium sections (grads). These match the dropdown options in your signup form: Pista Alta, Río Alto, and Marcador.

Data Security: All sensitive tokens and credentials are masked in this documentation. Use environment variables (.env) to store real credentials and never commit them to version control.
